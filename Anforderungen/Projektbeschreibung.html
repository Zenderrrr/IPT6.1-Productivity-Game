Projektbeschreibung

Aufgabenstellung

Entwickeln Sie eine Anwendung in C#, die das Konzept der objektorientierten Programmierung (OOP) verwendet und in einer dazugehörigen Datenbank ihre dazugehörigen Daten speichert. Als Datenbank gilt es SQL Lite zu verwenden.

  • Der gesamte Programmcode muss in einem Git Repository versionisiert verwaltet werden.
  • Die nötigen Schriftlichkeiten halten Sie mittels Markdown, ebenfalls im Git Repository fest.
  • Das Projekt soll in einem maximal 7 minütigen Video präsentiert werden.

Modulzuordnungen

Modul 162 «Daten analysieren und modellieren»

  • Erstellen Sie ein ER Modell Ihrer Datenbank und legen Sie es als Grafik im Git Repository ab
  • Erstellen Sie nach dem ER Modell das Relationale Modell und legen Sie es als Grafik im Git Repository ab

Modul 319 «Applikationen entwerfen und implementieren»

  • Entwickeln Sie eine Softwareanwendung und halten Sie die Funktionalität und das Ziel dieser Anwendung textuell fest.
  • Definieren Sie Features, welche Ihre Anwendung funktional abbilden soll.
  • Entwickeln Sie einen Programmablaufplan mittels PapDesigner, exportieren Sie den als Grafik und legen ihn als Grafik im Git Repository ab.
  • Halten Sie das alles in Ihrem Repository mittels Markdown fest

Modul 164 «Datenbanken erstellen und Daten einfügen»

  • Erstellen Sie ein SQL Script zum Erstellen der Datenbank, welches beim Starten der Anwendung prüft, ob die Datenbank bereits existiert. Wenn nicht, soll die Anwendung die Datenbank direkt erstellen
  • Erstellen Sie ein SQL Script zum Updaten der Datenbank, welches beim Starten der Anwendung ausgeführt wird, sobald ein solches Updatescript vorhanden ist. Nicht jede Version beinhaltet Datenbankänderungen, könnte aber sein.
  • Erstellen Sie ein SQL Script zum Einfügen von Beispieldaten in diese Datenbank. Erstellen Sie Funktionalität, die mittels Benutzerinteraktion Beispieldaten einfügt.

Modul 106 «Datenbanken abfragen, bearbeiten und warten»

  • Erstellen Sie Funktionalität, welche Datenbankzugriffe erfordern
  • Schreiben Sie Programmlogik nicht nur als Bestandteil der Software, sondern lagern Sie bewusst in Funktionen von ihrer Datenbank aus.
  • Schreiben Sie Tests zu diesen Datenbankfunktionalitäten

Modul 320 «Objektorientiert Programmieren»

  • Erstellung von Klassen und Objekten, die die Datenstruktur für die Datenbank abbilden.
  • Erstellen Sie dazu ein entsprechendes UML Klassendiagramm und legen Sie dieses als Grafik im Git Repository ab.
  • Implementierung von Vererbung, Polymorphismus und Abstraktion in den Klassen.
    • Implementieren Sie mindestens eine Klasse die von einer anderen Klasse Eigenschaften und Methoden erbt
    • Implementieren Sie ein Interface, welche zwei Methoden voraussetzt, welche zur Laufzeit unterschiedlich implementiert werden
    • Implementieren Sie eine Klasse, welche von einer abstrakten Klasse die Methode erbt, aber überschreibt

Allgemein

  • Die Anwendung soll Fehlerbehandlung und Validierung der Benutzereingaben enthalten
  • Die Anwendung soll eine benutzerfreundliche Oberfläche haben, die es dem Benutzer ermöglicht, Daten in der Datenbank zu persistieren
  • Die Persistierung der Daten erfolgt mittels SQL Lite. Dazu gehört der Verbindungsaufbau, die Datenbankerstellung, eine Datenbankupdateroutine, Datenbankabfragen und Datenmanipulationen
  • Implementierung von Methoden zur Datenbankabfrage und Datenbankmanipulation über die Datenbankverbindung
  • Es müssen jegliche CRUD Interaktionen mit der Datenbank abgedeckt werden
  • Vorgabe Software Design: Die Datenbankverbindung muss mittels Singleton Pattern implementiert werden
  • Funktionalitäten müssen mit Interfaces gewährleistet werden
  • Zusätzlich sollen Unit-Tests für die Klassen und Methoden implementiert werden, um die Funktionalität der Anwendung zu überprüfen
  • Die Anwendung soll in C# entwickelt werden und die Datenbankverwaltung soll mit SQL Lite realisiert werden.
